## 2. 转型理由

### v1 模式的核心问题（用户已判断）

| 维度 | v1 现状 | 是否可规模 |
|---|---|---|
| 端游攻略（手写）| 11 节、~5 万字、**3 次被纠错**（结局数、描述、合成配方）| ❌ 内容同质化、无护城河 |
| 仲魔公式（LLM 推测）| 50 个，**部分未验证** | ❌ 数据不可信 |
| 系统检测器 | 自动检测 + 多游戏，能跑 | ✅ **保留**（独立工具价值）|
| Pser_hanser 飞书 doc | 用户私人资料 | ✅ **保留**（个人 DB 雏形）|

### v2 模式的核心优势

1. **真护城河**：聚合 10+ 公开源 = 单游戏 ≥ 200 万字真实社区内容
2. **可规模化**：新游戏 = 索引 20 个新源（1 周）vs 手写 11 节攻略（1 月）
3. **数据驱动**：corpus 大到 RAG / 微调都有意义
4. **AI 摘要护城河**：别人只链原源不翻译，我翻译 = 唯一多语言层
5. **合规低风险**：只存摘要 + 链原源，不复制原文

## 6. 数据模型（重大修订）

### 6.1 `SourceSite` 实体（**新增**）

```typescript
{
  id: string;                // domain-based slug
  domain: string;            // "fandom.com" / "megamitensei.fandom.com"
  sourceName: string;        // "MegaTen Wiki"
  sourceType: "wiki" | "official" | "guide" | "personal_blog" | "rss";
  defaultLang: "en" | "ja" | "ko" | "zh";
  
  // 法规策略（每源独立）
  robotsUrl: string;
  tosUrl: string;
  licenseType: "cc-by-sa" | "cc-by" | "official" | "permission" | "unknown" | "restricted";
  crawlAllowed: boolean;
  crawlIntervalSec: number;  // 自定义限速, 不用 robots.txt 的 crawl-delay
  dailyLimit: number;
  contactEmail?: string;
  lastReviewedAt: ISODate;
  
  // 显示策略（驱动前端）
  displayPolicy: "metadata_only" | "summary" | "excerpt" | "full_translation";
  
  // 状态
  takedownStatus: "active" | "removed" | "blocked";
  notes?: string;
}
```

### 6.2 `Resource` 实体（修订）

```typescript
{
  id: string;
  sourceId: string;          // FK → SourceSite
  sourceUrl: string;
  sourceLang: "en" | "ja" | "ko" | "zh";
  gameSlug: string;
  topicTags: string[];
  title: { en, ja, ko, zh };     // 原 + AI 翻译
  author?: string;
  publishedAt?: ISODate;
  fetchedAt: ISODate;
  reliabilityScore: 1 | 2 | 3 | 4 | 5;
  keyTimestamps?: Array<{ time: number, label: string }>;
  
  // 内容（**DisplayPolicy 决定哪些字段对外可见**）
  summary: { en, ja, ko, zh };   // AI 摘要, always
  excerpt?: { en, ja, ko, zh };  // 短引用 < 200 字, optional
  
  // 内部字段（不对外展示）
  _internalHash?: string;        // dedup key
  _internalCanonicalUrl?: string;
}
```

**删除字段**（v0.1）：
- ❌ `content_snapshot` — 即使只缓存 50KB, 也是复制, 风险不可控

### 6.3 `Translation` 实体（修订）

```typescript
{
  resourceId: string;
  lang: "en" | "ja" | "ko" | "zh";
  title: string;
  summary: string;            // always
  excerpt?: string;           // optional, 由 DisplayPolicy 决定
  // full_text 默认不做
  translator: "ai-gpt4o" | "ai-claude" | "human";
  qualityScore: number;       // 0-1
  translatedAt: ISODate;
}
```

**删除字段**：默认无 `full_text`。仅当 `SourceSite.displayPolicy = "full_translation"` 且 `licenseType = "cc-by-sa" | "permission"` 时才生成，且必须显示 license 标注。

### 6.4 `IndexEntry` 实体（同 v0）

```typescript
{
  gameSlug: string;
  topicSlug: string;
  sourceResourceId: string;
  rankScore: number;          // 质量 × 时新 × 用户投票
  isFeatured: boolean;
}
```

### 6.5 `CrawlJob` 实体（**新增**）

```typescript
{
  sourceId: string;
  status: "queued" | "running" | "success" | "failed" | "blocked" | "circuit_broken";
  attempts: number;
  nextRunAt: ISODate;
  lastError?: string;
  startedAt?: ISODate;
  finishedAt?: ISODate;
}
```

## 7. DisplayPolicy（**新增**）

驱动前端显示行为的核心策略。

| policy | 标题 | AI 摘要 | 短引用 (< 200字) | 全文翻译 |
|---|---|---|---|---|
| `metadata_only` | ✅ | ❌ | ❌ | ❌ |
| `summary` | ✅ | ✅ | ❌ | ❌ |
| `excerpt` | ✅ | ✅ | ✅ | ❌ |
| `full_translation` | ✅ | ✅ | ✅ | ✅ (需 CC/授权) |

**默认策略**：`summary`（除非 license 是 official/cc-by-sa 才允许 `excerpt`，`full_translation` 必须有 `cc-by-sa | permission` + 强制 license 展示）。

**Outbound link 始终可见**——不依赖 canonical 解决版权问题。

## 11. 智能跳转规则（修订 v0.1）

```
用户访问 /zh/resource/{id}
  1. 查 resource.source_lang
  2. 查 resource.translations[zh]
  3. 查 SourceSite.displayPolicy
  
  IF source_lang === zh:
     → 显示原源（已是目标语言）
  
  ELSE IF translations[zh] 存在 AND displayPolicy ≥ "summary":
     → 显示我的 zh 摘要/翻译版
     → 顶部 banner: "本页面为 GameToolX AI 摘要/翻译，原文：[URL]"
     → 底部固定: "查看原文 →"（不靠 canonical 解决归属）
  
  ELSE:
     → 显示原源 (outbound)
     → 顶部 banner: "AI 摘要 (zh): ..." (1-2 句)
     → 按钮: "翻译此条"
```

**取消强制原文跳转**——`?lang=en` 不再 302 重定向到 en URL，改为显示"原源"链接让用户自行点击。

| L3 短引用（自动 50%）| ~$0.002 / 条 | $1 |
| L4 全文（用户触发 1%）| ~$0.02 / 次 | $0.2 |
| **小计** | | **<$10/月** |

## 13. 内容来源策略（修订 v0.1）

### 13.1 来源分类与策略

| 类型 | displayPolicy | legalType | 备注 |
|---|---|---|---|
| **公开 Wiki (Fandom/Miraheze)** | `excerpt` | `cc-by-sa` | 摘要 + 短引用, **强制显示 license** |
| **官方游戏站** | `metadata_only` | `official` | 仅标题/作者/时间, 摘要由我们自己写 |
| **官方攻略手册 / 资料集** | `metadata_only` | `official` | 同上 |
| **Steam 社区指南** | `summary` | `unknown` | 用户内容, 默认摘要 |
| **Game8 / GameWith / IGN** | `metadata_only` | `restricted` | **商业攻略站, 风险中高**, 仅元数据 + outbound link |
| **GameFAQs** | `summary` | `unknown` | 个人贡献, 摘要 |
| **个人博客（白名单）**| `summary` 或 `excerpt` | `permission` | 邮件授权后才允许 excerpt |
| **NGA / 贴吧 / B 站 / Reddit** | — | — | ❌ 完全不爬 |

### 13.2 爬取规范

1. **UA 透明**：`GameToolX-Bot/1.0 (+https://gametoolx.top/about)`
2. **robots.txt 严格遵守 allow/disallow**（仅这两个）
3. **不用 robots.txt 的 crawl-delay**（Google 官方明确说 Googlebot 不处理非标准 crawl-delay；Yandex/Bing 也不强制）—— 用自己的 per-domain token bucket
4. **每 source_id 独立令牌桶**，速率 = `crawlIntervalSec`
5. **每 source_id 每日 ≤ `dailyLimit` 请求**
6. **连续失败 3 次 → circuit breaker 24h**
7. **每域名每季度人工 review ToS**

## 14. 爬取工程（重大修订）

### 14.1 不用 robots.txt 的 crawl-delay

引用（用户验证）：
- Google 官方明确说 Googlebot 不处理 `crawl-delay` 指令（仅少数引擎如 Yandex/Baidu 处理）
- 即使使用，`crawl-delay` 是建议值，不是硬约束

**我们的策略**：每个 SourceSite 有自己的 token bucket，与 robots.txt 解耦。

### 14.2 Per-Domain 队列架构

```
[Scheduler]
   ↓ 每分钟扫描
   ↓ 选 nextRunAt <= now 的 sourceId
   ↓ 检查令牌桶 / daily limit / circuit breaker
[CrawlQueue (per sourceId)]
   ↓
[Worker Pool] (固定大小, e.g. 8 并发)
   ↓
[CrawlJob] → success / fail / circuit_broken
```

### 14.3 熔断规则

| 指标 | 阈值 | 处置 |
|---|---|---|
| 连续失败 | 3 次 | circuit_broken 24h |
| HTTP 429 | 1 次 | 退避 1h |
| HTTP 5xx 率 | > 50% (10 次内) | 暂停 6h |
| 抓取时间异常 | > 30s/req | 标记慢, 调整 rate |
| ToS 关键词命中 | 1 次 | 暂停 + 人工 review |
| robots.txt 新增 disallow 命中 | 1 次 | 立即停止该路径 |

### 14.4 Daily Limit 检查

每 source_id 每日 0:00 (UTC) 重置 `dailyLimit`。

## 15. Resource Dedup（**新增**）

```
去重策略 (从弱到强):
1. URL 完全匹配 → 直接合并
2. HTML <link rel="canonical"> 匹配 → 合并
3. 标题 embedding 相似度 ≥ 0.9 → 人工审核
4. _internalHash 匹配（内容指纹）→ 翻译缓存命中
5. 跨语言同源: 同一 hash 的不同 lang 版本 → 自动关联
```

翻译缓存 key = `_internalHash + lang`，所以同一内容翻译一次。

## 16. 编辑审核工作流（**新增**）

```
[Crawler 抓取]
   ↓
[Auto-cleanup]
   ├─ URL 已存在? skip
   ├─ canonical URL 已存在? skip
   ├─ 标题相似度 ≥ 0.9? skip
   └─ 通过 ↓
[AI Pipeline]
   ├─ 摘要生成 (quality < 0.6 → flag)
   ├─ 主题分类
   ├─ SourceSite.displayPolicy 检测
   └─ 通过 ↓
[Review Queue]
   ├─ displayPolicy = metadata_only / summary → auto-publish
   ├─ displayPolicy = excerpt / full_translation → human review
   └─ quality < 0.6 → human review

人工审核 (每周 1% 抽样):
  - 摘要准确性
  - 主题分类正确性
  - 源链接有效
  - 翻译质量 (如果有)
```

## 18. 合规与法律（重大修订）

### 18.1 Canonical 策略（分场景）

> Google 官方：canonical 是**强提示**，不一定保证按你想法收录。Google 用它来识别**重复/相似内容的代表 URL**。

| 页面类型 | displayPolicy | canonical | 原因 |
|---|---|---|---|
| 首页 | — | 自己 | 标准 |
| 游戏页 | — | 自己 | 聚合多源, 非镜像 |
| 主题页 | — | 自己 | 聚合多源, 非镜像 |
| 搜索页 | — | 自己 | 聚合多源, 非镜像 |
| 资源卡页 | `metadata_only` | 自己 | 非镜像 |
| 资源卡页 | `summary` | 自己 | 非镜像 |
| 资源卡页 | `excerpt` | 自己 | 短引用, 非镜像 |
| 资源卡页 | `full_translation` | **原源 或 noindex** | 近似镜像 |

**外链始终显眼**（不靠 canonical 解决版权归属）：

```
资源页底部固定：
"查看原文 →"           ← 出站链接, nofollow="ugc"
"本页面为 GameToolX 索引/摘要, 版权归原作者"
"原源 license: [CC BY-SA 4.0]"  (CC 源)
```

### 18.2 License 标注

CC BY-SA 源必须显示：
- License: CC BY-SA 4.0
- Author: [原作者]
- Source: [URL]
- License URL: https://creativecommons.org/licenses/by-sa/4.0/

CC BY-SA 的 ShareAlike 要求：衍生作品必须按相同或兼容 license 发布。我们的资源卡页 = 衍生作品 → 我们的页面也是 CC BY-SA。

### 18.3 DMCA / 投诉流程

- `/dmca` 表单
- 作者/权利人 24h 内响应
- 投诉核实后 1 小时内移除 + 加入黑名单

### 18.4 不复制全文原则

- 不存 `content_snapshot`
- 不存 `full_text`（默认）
- 不做"内容镜像站"

## 25. 阶段路线（重大修订）

### Phase 1（6-8 周，**拆 4 子阶段**）

#### P1a（1-2 周）：Source registry + 手工录入

- 数据模型（SourceSite + Resource + IndexEntry）
- Admin UI：手工录入资源
- 基础 UI：资源卡 + 主题过滤
- **目标**：2 个游戏 × 25 条 = **50 条手工资源**
- **不写 crawler**

#### P1b（2-3 周）：Crawler + Per-Domain 队列

- CrawlJob 队列
- Per-domain token bucket
- 失败熔断 (circuit breaker)
- robots.txt 解析（仅 allow/disallow）
- DisplayPolicy 自动判定
- **目标**：4 个域名 × 30 条 = **120 条自动抓**

#### P1c（1 周）：AI 摘要 / 翻译缓存

- 摘要管线 (GPT-4o-mini)
- 关键点提取 (Claude Sonnet)
- 术语表锁定
- 翻译缓存 (按 _internalHash + lang)

#### P1d（1 周）：主题页 + sitemap + canonical 实施

- 主题分类路由
- 多语言 sitemap.xml
- hreflang + canonical 分场景
- DMCA 页面

**合计 Phase 1 = 6-8 周**（比原 4 周翻倍，但更现实）

### Phase 2（4-6 周，资源库 ≥ 1000 后启动）

- Vector search（sqlite-vss）
- RAG Q&A（免费 5 次/日 + 订阅无限）
- 内容来源扩张到 5 个游戏
- 联盟链接（Steam 史低）

### Phase 3（v1.5+）

- B2B API（结构化 JSON）
- 与游戏媒体 / 发行商合作
- 海外扩展（zh-TW / vi / th / id）

## 26. 风险与缓解（修订 v0.1）

| 风险 | 等级 | 缓解 |
|---|---|---|
| **CC BY-SA ShareAlike 传染** | 中 | 资源卡页 = 衍生作品, 我们的页面也需 CC BY-SA; 在 footer 显示 |
| **DMCA / 律师函** | 中 | 白名单 + DisplayPolicy + 不复制全文 + 24h 响应 |
| **Google 判重复内容** | 中 | canonical 分场景, summary 级别不指原源 |
| **商业攻略站 (Game8) 投诉** | 中 | displayPolicy = metadata_only, 仅链出, 不抓内容 |
| **RAG Q&A 编造** | 高 | 推迟 RAG, 等资源库 ≥ 1000 条且质量稳定后才做 |
| **冷启动无用户** | 高 | SEO 长尾（6 个月起量）|
| **翻译质量差** | 中 | 术语锁定 + 月度人工抽查 + 置信度 badge |
| **LLM 成本失控** | 低 | 缓存 + 限速 + 不默认全文翻译 |
| **单点故障** | 中 | 备份到另一家云 |
